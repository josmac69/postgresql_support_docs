# PostgreSQL 18 (REL_18_STABLE) — Comprehensive Shared Memory Object Table

All objects allocated in the main shared memory segment, as summed by `CalculateShmemSize()` in `src/backend/storage/ipc/ipci.c` and initialized by `CreateOrAttachShmemStructs()`. Names in the **Shmem name** column are the `ShmemInitStruct()` registration names visible in `pg_shmem_allocations` (— means internal / no separate named entry).

## Abbreviations & Terminology

| Abbreviation | Meaning |
|---|---|
| **AIO** | Asynchronous I/O — the new PG18 subsystem that issues disk reads without blocking the backend |
| **bgwriter** | Background Writer — auxiliary process that pre-cleans dirty buffers ahead of the clock sweep |
| **BLCKSZ** | Block Size — size of one data page, 8 KB by default |
| **CLOG** | Commit Log (directory `pg_xact`) — per-transaction commit/abort status |
| **CV** | Condition Variable — sleep/wake primitive; a process sleeps until another broadcasts an event |
| **DSA** | Dynamic Shared Area — a malloc-style allocator built on top of DSM segments |
| **DSM** | Dynamic Shared Memory — segments created at runtime (e.g. for parallel query), separate from the main segment |
| **dshash** | Dynamic Shared Hash table — expandable hash table built on a DSA |
| **fsync** | File synchronization — forcing written data from OS cache to durable storage |
| **GUC** | Grand Unified Configuration — a PostgreSQL configuration parameter (`postgresql.conf` setting) |
| **GXACT** | Global Transaction — an entry describing a prepared (two-phase) transaction |
| **iovec** | I/O vector — OS structure describing one chunk of a scatter/gather (multi-buffer) I/O operation |
| **io_uring** | Linux kernel interface (5.1+) for submitting/completing async I/O via shared rings with minimal syscalls |
| **IPC** | Inter-Process Communication |
| **Latch** | PostgreSQL's inter-process wakeup primitive (backed by signals + a self-pipe/epoll) |
| **LSN** | Log Sequence Number — a byte position in the WAL stream; identifies a WAL record |
| **LWLock** | Lightweight Lock — fast shared/exclusive lock protecting shared memory structures (not user data rows) |
| **MultiXact** | Multi-transaction — an ID representing *multiple* XIDs jointly locking one row (e.g. several `FOR SHARE` lockers) |
| **MVCC** | Multi-Version Concurrency Control — PostgreSQL's approach of keeping multiple row versions instead of read locks |
| **NUMA** | Non-Uniform Memory Access — multi-socket architecture where memory latency depends on which node owns the page |
| **OID** | Object Identifier — internal numeric identifier of database objects |
| **PGPROC** | Per-process shared struct — one per backend/auxiliary process; the process's "identity card" in shared memory |
| **PROC_HDR** | Process Header (`ProcGlobal`) — the global bookkeeping struct managing all PGPROC entries |
| **SLRU** | Simple Least-Recently-Used — small page-cache mechanism for on-disk ring-structured metadata (CLOG, subtrans, multixact, …) |
| **SSI** | Serializable Snapshot Isolation — the algorithm behind `SERIALIZABLE` isolation; tracks read/write dependencies via predicate locks |
| **sinval** | Shared Invalidation — messages telling other backends "your cached catalog/relation info is stale" |
| **Tranche** | A named group of related LWLocks (what you see in `pg_stat_activity.wait_event` for `LWLock` waits) |
| **WAL** | Write-Ahead Log (a.k.a. xlog) — the redo log; every change is logged before the data page is written |
| **XID** | Transaction ID — 32-bit transaction identifier subject to wraparound management |
| **2PC** | Two-Phase Commit — `PREPARE TRANSACTION` / `COMMIT PREPARED` for external transaction managers |

## The Object Table

| # | Category | Object / Shmem name | Source file(s) | Size function | Sizing GUCs / formula | Purpose (technical) | Used for (plain terms) |
|---|----------|--------------------|----------------|---------------|----------------------|---------------------|------------------------|
| 1 | Core | **ShmemIndex** (`ShmemIndex`) | `storage/ipc/shmem.c` | `hash_estimate_size(SHMEM_INDEX_SIZE=64, ...)` | fixed | Hash table mapping structure names → address/size; backs `pg_shmem_allocations` | The "table of contents" of shared memory: lets every process find each named structure by name |
| 2 | Core | **DSM control** | `storage/ipc/dsm.c` | `dsm_estimate_size()` | `min_dynamic_shared_memory` | Bookkeeping for dynamic shared memory segments + optional pre-reserved DSM pool | Tracks which runtime-created memory segments exist (parallel query workers attach through this) |
| 3 | Core | **DSM Registry** (`DSM Registry Data`) | `storage/ipc/dsm_registry.c` | `DSMRegistryShmemSize()` | fixed | Name→handle dshash for `GetNamedDSMSegment()` / `GetNamedDSA()` / `GetNamedDSHash()` (PG17+, extended in PG18) | Phone book letting extensions find/create shared segments by name without `shared_preload_libraries` |
| 4 | Buffers | **Buffer Blocks** (`Buffer Blocks`) | `storage/buffer/buf_init.c` | `BufferManagerShmemSize()` | `NBuffers × BLCKSZ + PG_IO_ALIGN_SIZE` (= `shared_buffers`) | The actual 8 KB page frames — the buffer pool itself | The database page cache: table and index pages are read into and modified in these frames |
| 5 | Buffers | **Buffer Descriptors** (`Buffer Descriptors`) | `storage/buffer/buf_init.c` | part of `BufferManagerShmemSize()` | `NBuffers × sizeof(BufferDescPadded)` (64 B each) | Per-buffer header: `BufferTag`, atomic state word (refcount / usage count / flags), embedded content LWLock, AIO write-reference (PG18) | The metadata card for each cached page: which page it holds, who's using it, is it dirty, is I/O in progress |
| 6 | Buffers | **Buffer IO condition variables** (`Buffer IO Condition Variables`) | `storage/buffer/buf_init.c` | part of `BufferManagerShmemSize()` | `NBuffers × sizeof(ConditionVariableMinimallyPadded)` | CVs backends wait on for in-progress buffer IO to complete (replaced the old per-buffer IO LWLocks) | The waiting room: if another process is already reading a page from disk, you sleep here until it's done |
| 7 | Buffers | **Checkpoint BufferIds** (`Checkpoint BufferIds`) | `storage/buffer/buf_init.c` | part of `BufferManagerShmemSize()` | `NBuffers × sizeof(CkptSortItem)` | `CkptBufferIds`: array the checkpointer uses to sort dirty buffers by relfile/fork/block for sequential writeback | Scratchpad so checkpoints write dirty pages in on-disk order (sequential I/O) instead of randomly |
| 8 | Buffers | **Strategy control** (`Buffer Strategy Status`) | `storage/buffer/freelist.c` | `StrategyShmemSize()` | fixed (`sizeof(BufferStrategyControl)`) | Clock-sweep hand (`nextVictimBuffer`), freelist head/tail, bgwriter scan position, completed-passes / alloc counters | The eviction controller: decides which cached page gets thrown out when a new one needs space |
| 9 | Buffers | **Buffer mapping hash** (`Shared Buffer Lookup Table`) | `storage/buffer/buf_table.c` | `BufTableShmemSize()` | `hash_estimate_size(NBuffers + NUM_BUFFER_PARTITIONS(128))` | `BufferTag` → buffer id lookup, striped across 128 buffer-mapping LWLock partitions | The index of the cache: answers "is page X of table Y already in memory, and in which frame?" |
| 10 | WAL | **XLogCtl** (`XLOG Ctl`) | `access/transam/xlog.c` | `XLOGShmemSize()` | `wal_buffers` (auto = `shared_buffers`/32); `sizeof(XLogCtlData)` + 9 × `WALInsertLockPadded` + `XLOGbuffers × sizeof(XLogRecPtr)` + `XLOGbuffers × XLOG_BLCKSZ` | Central WAL state: insert/write/flush positions, `LogwrtResult`, timeline info, 8 WAL insertion locks, `xlblocks` array, and the WAL buffers themselves | The heart of durability: staging area where every change record is assembled before being flushed to the WAL files at commit |
| 11 | WAL | **XLog prefetch stats** (`XLogPrefetchStats`) | `access/transam/xlogprefetcher.c` | `XLogPrefetchShmemSize()` | fixed | Recovery prefetching counters (`pg_stat_recovery_prefetch`) | Counters showing how well recovery/replay pre-reads the data pages it's about to need |
| 12 | WAL | **XLog recovery state** (`XLOG Recovery Ctl`) | `access/transam/xlogrecovery.c` | `XLogRecoveryShmemSize()` | fixed | Standby/recovery shared state: replay position, recovery pause, promotion handling | Shared status of crash recovery / standby replay: how far replay has gotten, is it paused, is promotion requested |
| 13 | WAL | **WAL summarizer** (`WalSummarizerCtl`) | `postmaster/walsummarizer.c` | `WalSummarizerShmemSize()` | fixed; active with `summarize_wal=on` | WAL summarizer process state for incremental backup (PG17+) | Coordinates the process that condenses WAL into block-level summaries so `pg_basebackup --incremental` knows what changed |
| 14 | Transam | **TransamVariables** (`TransamVariables`) | `access/transam/varsup.c` | `VarsupShmemSize()` | fixed | Shared XID/OID counters: `nextXid`, `oldestXid`, `xidVacLimit`, `nextOid`, wraparound limits (historically `VariableCache`) | The ticket dispenser: hands out the next transaction ID and OID, and tracks wraparound danger thresholds |
| 15 | SLRU | **CLOG / pg_xact** (`transaction` SLRU) | `access/transam/clog.c`, `slru.c` | `CLOGShmemSize()` | `transaction_buffers` (0 = auto from `shared_buffers`) | Commit/abort status, 2 bits per transaction | The commit ledger: MVCC visibility checks ask it "did transaction N commit or abort?" |
| 16 | SLRU | **Subtrans** (`subtransaction` SLRU) | `access/transam/subtrans.c` | `SUBTRANSShmemSize()` | `subtransaction_buffers` (0 = auto) | Subtransaction XID → parent XID mapping | Family tree of savepoints: maps each subtransaction to its parent so visibility rules can climb to the top-level XID |
| 17 | SLRU | **MultiXact offsets** (`multixact_offset` SLRU) | `access/transam/multixact.c` | part of `MultiXactShmemSize()` | `multixact_offset_buffers` (default 16) | MultiXact ID → members-array offset mapping | Lookup step 1 for shared row locks: where in the members area does this MultiXact's list start? |
| 18 | SLRU | **MultiXact members** (`multixact_member` SLRU) + `MultiXactState` | `access/transam/multixact.c` | part of `MultiXactShmemSize()` | `multixact_member_buffers` (default 32) | MultiXact member arrays (locker XIDs + lock modes) + shared generation state, per-backend `OldestMemberMXactId` array | Lookup step 2: the actual list of transactions jointly holding a row lock (e.g. several `SELECT … FOR SHARE`) |
| 19 | SLRU | **Commit timestamps** (`commit_timestamp` SLRU + `CommitTs shared`) | `access/transam/commit_ts.c` | `CommitTsShmemSize()` | `commit_timestamp_buffers` (0 = auto); active with `track_commit_timestamp` | Per-transaction commit timestamps (+ optional replication origin id) | Answers "when exactly did transaction N commit?" — needed by some replication/conflict-resolution setups |
| 20 | SLRU | **Serial / pg_serial** (`serializable` SLRU) | `storage/lmgr/predicate.c` | part of `PredicateLockShmemSize()` | `serializable_buffers` (default 32) | SSI: conflict data for old committed serializable transactions | Long-term memory of SSI: keeps conflict info about committed `SERIALIZABLE` transactions that still matter |
| 21 | SLRU | **Notify queue** (`notify` SLRU) + **AsyncQueueControl** (`Async Queue Control`) | `commands/async.c` | `AsyncShmemSize()` | `notify_buffers` (default 16); control struct scales with `MaxBackends` | LISTEN/NOTIFY message ring (SLRU pages) + head/tail pointers and per-backend read positions | The mailbox for `NOTIFY` messages: senders append, each `LISTEN`ing backend reads at its own pace |
| 22 | Transam | **TwoPhaseState** (`Prepared Transaction Table`) | `access/transam/twophase.c` | `TwoPhaseShmemSize()` | `max_prepared_transactions` × `GlobalTransactionData` | Prepared (2PC) transaction slots incl. dummy PGPROC-linked GXACT entries | Holds transactions frozen by `PREPARE TRANSACTION`, keeping their locks/XID alive until `COMMIT/ROLLBACK PREPARED` |
| 23 | Procs | **ProcArray** (`Proc Array`) + **KnownAssignedXids** | `storage/ipc/procarray.c` | `ProcArrayShmemSize()` | `PROCARRAY_MAXPROCS = MaxBackends + max_prepared_transactions`; KnownAssignedXids = `TOTAL_MAX_CACHED_SUBXIDS` (standby only) | Array of active pgprocnos for snapshot computation; standby-side in-progress-XID tracking for Hot Standby | The roll call of running transactions — scanned every time a snapshot is taken ("who is in flight right now?") |
| 24 | Procs | **ProcGlobal / PGPROC array** (`Proc Header`, PGPROC structures, per-proc semaphores) | `storage/lmgr/proc.c` | `ProcGlobalShmemSize()` | `TotalProcs = MaxBackends + NUM_AUXILIARY_PROCS + max_prepared_transactions`; one semaphore per proc | `PROC_HDR` free lists + per-process `PGPROC`: XID, LSN, wait state, subxid cache, group-commit links, fast-path pointers | Each process's shared identity card: its XID, what it's waiting for, and the semaphore used to put it to sleep/wake it |
| 25 | Locks | **Fast-Path Lock Array** (`Fast-Path Lock Array`) — *new in PG18* | `storage/lmgr/proc.c`, `utils/init/postinit.c` | `FastPathLockShmemSize()` | `TotalProcs × (FastPathLockGroupsPerBackend × 8 B + slots × sizeof(Oid))`; groups derived from `max_locks_per_transaction` (16 slots/group, up to 1024 groups) | Variable-sized per-backend fast-path relation lock slots (was fixed 16 inline in PGPROC before 18); reduces `LWLock:LockManager` contention | Express checkout for common table locks: weak locks recorded privately per backend, skipping the shared lock table |
| 26 | Locks | **LOCK hash** (`LOCK hash`) | `storage/lmgr/lock.c` | part of `LockManagerShmemSize()` | `NLOCKENTS() = max_locks_per_transaction × (MaxBackends + max_prepared_transactions)`; 16 partitions | Heavyweight lockable-object table: one entry per locked object with grant/wait state | The main lock table behind `pg_locks`: one entry per locked table/row-range/object, with who holds and who waits |

## Verify on a live instance

```sql
SELECT name, off, size, pg_size_pretty(size)
FROM pg_shmem_allocations
ORDER BY size DESC;

-- PG18, Linux, built --with-libnuma (expensive — touches every page):
SELECT * FROM pg_shmem_allocations_numa;
```

*Note: exact `ShmemInitStruct` name strings can differ slightly between versions; treat the names above as the PG18-era identifiers and confirm against `pg_shmem_allocations` on your build. Sizing formulas are structural — alignment (`MAXALIGN`, cacheline padding, `PG_IO_ALIGN_SIZE`) and safety margins add real bytes on top.*
